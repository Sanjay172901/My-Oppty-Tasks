19. HTML Comments

Comments are used to explain HTML code and improve readability. They are completely ignored by the browser and are not visible to users. Comments are essential for documentation, collaboration, and debugging.

Comment Syntax

HTML comments begin with <!-- and end with -->. Everything inside is ignored by the browser. 

<!-- This is a comment -->
<p>This is visible content.</p>

Single-Line and Multi-Line Comments

<!-- Single line comment -->

<!--
Multi-line comment
that spans several lines
-->

Comment Placement

In the Head Section

<head>
  <!-- SEO Meta Tags -->
  <meta name="description" content="Page description">

  <!-- External CSS -->
  <link rel="stylesheet" href="styles.css">
</head>

In the Body Section

<body>
  <!-- Header Section -->
  <header>
    <h1>Welcome</h1>
  </header>
</body>

Commenting Out Code

Comments can temporarily disable HTML elements during testing. 

<div>
  <h2>Visible Heading</h2>

  <!-- <h2>Hidden Heading</h2> -->

  <p>This paragraph is visible.</p>
</div>

Best Practices for Comments

  • Explain complex logic
  • Document important decisions
  • Mark sections clearly
  • Keep comments updated
  • Do not over-comment obvious code

Marking Sections Clearly

<!-- ========== HEADER SECTION ========== -->
<header></header>

<!-- ========== MAIN CONTENT ========== -->
<main></main>

<!-- ========== FOOTER SECTION ========== -->
<footer></footer>

Comments in Forms

<form>
  <!-- User Info -->
  <input type="text">

  <!-- Submit Button -->
  <input type="submit">
</form>

JavaScript vs HTML Comments

HTML comments use <!-- -->. JavaScript comments use:

  • // Single line
  • /* Multi-line */
<script>
// Single line JS comment

/*
Multi-line
JS comment
*/
</script>

Complete Example

<!DOCTYPE html>
<html>

<!-- Author: John Doe -->
<!-- Purpose: Demonstrate HTML Comments -->

<head>
  <!-- Page Title -->
  <title>Comment Example</title>
</head>

<body>

  <!-- Header Section -->
  <h1>Welcome</h1>

  <!-- Main Paragraph -->
  <p>This page demonstrates comments.</p>

</body>
</html>
