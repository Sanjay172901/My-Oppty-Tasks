18. Input Types & Validations

HTML forms collect user data. Input validation ensures that the data entered is correct, complete, and secure before submission. Modern HTML5 provides built-in validation features without JavaScript.

What is Input Validation?

Input validation ensures:

  • Required fields are not empty
  • Email format is correct
  • Numbers stay within allowed ranges
  • Data matches required patterns
  • Files meet type restrictions

Validation improves security, user experience, and data accuracy.

Common Text Input Types

<input type="text" name="username" required>

<input type="email" name="email" required>

<input type="password" name="password" required>

<input type="tel" name="phone">

<input type="url" name="website">

Numeric Validation

Use min, max, and step to control numeric input ranges. 

<input type="number" name="age" min="1" max="120">

<input type="range" min="0" max="100">

Pattern Validation

The pattern attribute uses Regular Expressions (Regex) to validate user input. 

<input type="text" name="code"
pattern="[A-Za-z]{3}"
title="Enter exactly three letters">

File Validation

The accept attribute restricts file types. 

<input type="file" accept="image/*">

Best Practices for Validation

  • Always use required for important fields
  • Use semantic input types (email, number, date)
  • Restrict numeric ranges
  • Validate on both client-side and server-side
  • Provide clear error messages

Client-Side vs Server-Side Validation

Client-side validation (HTML/JS) improves user experience. Server-side validation ensures security and must always be implemented. Never rely only on frontend validation.
