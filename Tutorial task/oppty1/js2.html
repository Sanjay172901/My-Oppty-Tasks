2. JavaScript Syntax

Master the rules and structure of JavaScript statements

A computer program is a list of "instructions" to be executed by a computer. In programming, these instructions are called statements.

// Example Syntax
let x = 5; let y = 6; 
let z = x + y;
console.log(z); // Output: 11

Semicolons

Semicolons separate JavaScript statements. It is good practice to end statements with them, though JavaScript has automatic semicolon insertion.

let a = 1; let b = 2; let c = a + b;

Case Sensitivity & Identifiers

JavaScript is case-sensitive. This means that variable names like lastName and lastname are considered two completely different variables.

Camel Case Convention

While you can name variables in many ways, JavaScript developers historically prefer lowerCamelCase. The first word is lowercase, and each subsequent word starts with a capital letter (e.g., userFirstName, totalPrice).

JavaScript Keywords

Keywords are reserved words used to identify the action to be performed. They cannot be used as variable names:

  • var, let, const: Declare variables
  • if, else, switch: Control flow logic
  • for, while, do: Loop structures
  • function: Define reusable logic
  • return: Exit function and return value
  • try, catch: Handle errors

Whitespace & Comments

JavaScript ignores multiple spaces. You can add whitespace to your script to make it more readable.

// Good readability
let person = "He-Man";
let space  = "Eternia";

// Comments explain code
// This is a single line comment
/* This is a 
   multi-line comment */

Variables & Data Types

Variables store data values, while Data Types define the kind of data being stored:

// Variables
const pi = 3.14; 
let name = "John"; 

// Data Types
let age = 25;           // Number
let message = "Hello";  // String
let isActive = true;    // Boolean
let data = null;        // Null

Functions

Functions are blocks of code designed to perform a particular task, executed when "called":

function greet(user) {
  return "Welcome, " + user + "!";
}
console.log(greet("Developer"));
