9. Multimedia (Audio & Video)

HTML5 introduced built-in support for multimedia elements such as audio and video. Before HTML5, multimedia required plugins like Flash. Now, browsers can directly play media using the <audio> and <video> tags.

Adding Audio

The <audio> tag embeds sound content in a webpage. The controls attribute adds play, pause, and volume controls. 

<audio controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

Output

The <source> tag allows multiple audio formats to ensure browser compatibility. If the browser does not support the audio element, fallback text is displayed.

Important Audio Attributes

  • controls – Displays playback controls.
  • autoplay – Starts playing automatically.
  • loop – Repeats audio continuously.
  • muted – Starts audio muted.

Adding Video

The <video> tag is used to embed video files. You can specify width and height to control the display size. 

<video width="320" height="240" controls>
  <source src="https://www.w3schools.com/html/mov_bbb.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

Output

Important Video Attributes

  • controls – Shows video controls.
  • autoplay – Plays automatically.
  • loop – Repeats the video.
  • muted – Starts muted.
  • poster – Displays preview image before playing.

Using Poster Attribute

<video width="320" controls 
poster="https://via.placeholder.com/320x180">
  <source src="https://www.w3schools.com/html/mov_bbb.mp4" type="video/mp4">
</video>

Output

Why Multimedia is Important

  • Improves user engagement
  • Enhances learning experience
  • Supports modern web applications
  • Eliminates need for external plugins

Best Practices

  • Always include fallback text.
  • Provide multiple formats (MP4, WebM, MP3).
  • Use compressed media for faster loading.
  • Avoid autoplay for better user experience.
