16. API Integration

Connect your applications to external data sources using Fetch and Async logic

What is an API?

An Application Programming Interface (API) is a set of rules that allows one software application to interact with another. In web development, we typically use "Web APIs" to fetch data from a server in JSON format.

JSON (JavaScript Object Notation): A lightweight data-interchange format that is easy for humans to read and easy for machines to parse.

The Fetch API

The fetch() method is modern JavaScript's way of making network requests. It returns a Promise that resolves to the Response object.

// Basic Fetch Syntax
fetch('https://api.example.com/data')
  .then(response => response.json()) // Convert to JSON
  .then(data => console.log(data))   // Use data
  .catch(error => console.error(error)); // Handle errors

Async and Await

Async/Await is the modern, more readable way to handle asynchronous operations. It allows you to write asynchronous code that looks and behaves like synchronous code.

async function getData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.log("Something went wrong:", error);
  }
}

The async keyword makes the function return a promise. The await keyword makes JavaScript wait until the promise settles and returns its result.

HTTP Methods

While GET is the default for fetching data, APIs support other methods:

  • GET: Request data from a resource.
  • POST: Submit data to be processed to a resource.
  • PUT: Update an existing resource.
  • DELETE: Remove a resource.
